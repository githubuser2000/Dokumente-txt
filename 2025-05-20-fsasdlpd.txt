ALX: [Ontological file system through semantics (invention)]

What's not there
• Folder hierarchy, not a directory tree.
• Files, instead a box or an encapsulation or capsule, inside and outside of which everything and nothing could or can be, thus defined extremely arbitrarily.
• File names probably not even yet, but anything else that could replace file names, i.e., all forms of data that exist today could replace text file names. File names will certainly exist as a possibility, as a form of identifier, which can be not just text or a name, but almost anything: even an identifier or ID can be omitted.

Today's file systems are:
• Hierarchies (12)
• Graphs or networks (24) through symlinks and hardlinks
• Some other things

What do you need to know to master ontological file systems?
• Elementary school is no longer enough; it's not until your mid-20s that you're smart enough to use this far too complicated file system (sarcasm)
• Yes, but it's really bad that when things get better, everything just keeps getting more complicated.

Example 1
• Assume there is a file named filename.tar.xz.gpg
• Today, these are simply characters and strings to which we mentally assign a meaning. Furthermore, within the file, there's an additional detection to determine whether it's actually in this format.
• In an ontological file system, these would be concatenations of meaning assignments, so tar xz and gpg would simply be such ontological concatenations.
• And this concatenation is then also linked to the software that can load this file.
• For example, instead of the .mp3 extension, there is ontological information that links the mp3 to the media player.
